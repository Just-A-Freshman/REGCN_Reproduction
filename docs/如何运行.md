## 二、配置文件（`REGCN/config.ini`）

```ini
[hyper]
datasets = SSE                          # SSE 或 DJIA
adj_type2 = _90.npy                    # 邻接矩阵阈值后缀
r_mse = 0.0                            # L1 正则化系数（必须 0.0）
r_acc = 0.1                            # 趋势准确率正则化系数
train_rate = 0.7                       # 训练集比例
val_rate = 0.1                         # 验证集比例
seq_len = 30                           # 滑动窗口长度
label_col = 3                          # 收盘价列索引
data_addr = ../data/data/              # 原始数据路径
adj_addr = ../data/adj/                # 邻接矩阵路径
all_data = 0                           # 1=全部股票，0=单支
start_index = 0                        # 起始股票索引（全量模式）
s_index = 0                            # 股票索引（单支模式）
lr = 8e-3                              # 学习率
n_neurons = 128                        # GRU 隐藏单元数
n_epochs = 100                         # 训练轮数
batch_size = 128                       # 批次大小
vmd_addr = ../data/data/VMDdata/       # VMD 数据路径
n_jobs = 1                             # 并行数（>1 启用多进程）
```

**关键说明：** `r_mse=0.0` 是必须的。若 r_mse > 0，L1 正则化会压塌模型，导致常数输出。

## 五、图表生成

### 5.1 Fig 3 — 预测对比图

从已保存的模型权重（`model/` 目录下的 `.weights.h5` 文件）直接生成预测对比图，无需重新训练。

```bash
# SSE 全部 40 支
python fig3_prediction_plots.py --datasets SSE

# DJIA 全部 28 支
python fig3_prediction_plots.py --datasets DJIA

# 指定范围
python fig3_prediction_plots.py --datasets SSE --start 0 --end 5
```

输出：`result/Fig3/REGCN_{SSE,DJIA}-{index}.png`（共 68 张）

**注意：** 模型权重文件路径必须为 `model/model_{DATASET}-{INDEX}-comp{N}.weights.h5`。
Windows 下 h5py 无法处理中文路径，需将权重复制到纯英文路径（如 `C:\temp\stock_models\`），或在 Linux 上运行。

### 5.2 Fig 4 — 正则化系数 Grid Scan

扫描 r_mse × r_acc 的 4×5=20 种组合，在单支股票上评估性能。

```bash
python fig4_regularization_scan.py
```

默认在 SSE Stock 0 上运行。结果保存到 `result/Figure4/figure4_stock0_grid.csv`。

可通过修改脚本中的 `r_mse_values` 和 `r_acc_values` 数组调整扫描范围。

### 5.3 Table 6 — 消融实验

测试 6 种消融变体（Methods 5-10）在全量 SSE 和 DJIA 上的性能。

```bash
# 单支股票调试
python table6_ablation.py --method 6 --datasets SSE --start 0 --end 1

# 全量 SSE + DJIA（串行）
python table6_ablation.py --method 6 --datasets SSE DJIA

# 全量 + 16 路并行
python table6_ablation.py --method 6 --datasets SSE DJIA --workers 16
```

支持的 method 值：
| Method | 名称 | 说明 |
|:------:|------|------|
| 5 | 无分解 | 跳过 VMD，用原始数据 |
| 6 | 单图 Pearson | 仅保留 Pearson 相关系数图 |
| 7 | 单图 Spearman | 仅保留 Spearman 相关系数图 |
| 8 | 单图 DTW | 仅保留 DTW 距离图 |
| 9 | 无 GCN | GRU 代替 GCGRU cell |
| 10 | MSE loss | 去掉趋势正则化 |

输出：`result/Table6/{method_name}/{dataset}_{method}.csv`

**重要：** 脚本通过外部参数控制，**不修改 REGCN/ 下的任何原始代码**。每个 method 通过修改传入 `trainmodel()` 的数据（替换邻接矩阵、跳过 VMD 循环等）来实现消融。

## 七、常见问题

### Q1: 模型输出是常数，R² 为负？

**原因：** `r_mse` 太大，L1 正则化压塌了模型。
**解决：** 确保 `config.ini` 中 `r_mse = 0.0`。

### Q2: 预测对比图是常数（蓝线水平）？

**原因：** 该股票模型训练失败，常见于 SSE stock 6/10/29/34 等"灾难性失败"股票。
**说明：** 这些股票在原始全量训练中也表现差（R² 为负），并非权重加载问题。

### Q3: h5py 无法打开权重文件？

**原因：** Windows 下 h5py 不支持中文路径。
**解决：** 将 `model/` 目录复制到纯英文路径（如 `C:\temp\stock_models\`），或用 Linux 运行。

### Q4: 哪些文件是临时文件可以删除？

| 文件 | 状态 | 说明 |
|------|:----:|------|
| `sync_and_run_djia_ablation.py` | 已删 | 一次性云同步脚本 |
| `run_djia_ablation.py` | 已删 | 被 `table6_ablation.py` 替代 |
| `bash.exe.stackdump` | 已删 | 崩溃转储 |
| `model/*.weights.h5` | 保留 | Fig 3 生成需要（如磁盘不足可删，删除后需重新训练） |
