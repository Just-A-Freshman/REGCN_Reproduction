## 1.3 批量归一化

批量归一化的方法使用“最大最小值”归一化。以开盘价字段为例，我们先求出训练数据集中，最小的开盘价和最大的开盘价，然后对所有开盘价进行归一化操作：
$$
归一化开盘价 = \frac{开盘价 - 最小开盘价}{最大开盘价 - 最小开盘价}
$$
注意：将归一化后的测试数据集输入模型后，需要反归一化才能拿到最终的可用的预测结果。

补充，数据有一点比较非常巧合的地方：

- **原始数据**中，DJIA有754行，其中第一行是表头，还有753行才是数据。但因为股票日期没有对齐。对齐后，28 支股票取共同交易日，最终得到752天的数据；
- 原始代码中给出的**归一化后的数据**，也是754行。经过检查发现，这是因为，处理后的数据没了表头，因此真实数据的确有754行。但最后两行数据是每个字段原始数据的最大值和最小值（用于模型预测后的反归一化），减去它们就得到了752天数据。



## 1.4 滑动窗口构建输入和输出

时间序列预测里有一个基本约束：**你不能拿未来的数据预测过去**。所以不能用第 50-100 天训练、第 1-49 天测试，这没有意义。滑动窗口的做法是：**每次用过去 seq_len=30 天，预测第 31 天**。这样，如果原本有32天的数据，我们就只能构造出2条[输入 -> 输出]的数据。而我们实际上有训练601条，因此可以构建：$601 - 30 = 571$个[输入->输出]的训练样本，构建：$151 - 30 = 121$个测试样本。一个DJIA的样本belike：

```

┌────────┬──────┬──────┬──────┬──────┬──────┬──────┐   ┌────────┐
│ day    │ open │ close│ high │ low  │ vol  │ ...  │   │ close  │
├────────┼──────┼──────┼──────┼──────┼──────┼──────┤   ├────────┤
│ i      │ v    │ v    │ v    │ v    │ v    │ v    │   │ i+1    │
│ i+1    │ v    │ v    │ v    │ v    │ v    │ v    │   │ i+2    │
│ ...    │ ...  │ ...  │ ...  │ ...  │ ...  │ ...  │   │ ...    │
```

# 2. VMD拆分

## 2.1 GA遗传算法确定最优超参数

使用GA遗传算法确定超参数$K$和$\alpha$，固定参数`tau=0`（噪声容限）、`DC=0`（序列不含常量）、`init=1`（均匀初始化中心频率）、`tol=1e-7`（收敛容差）：

- **K**：分解的模态数（IMF 个数），范围 2-5
- **α**：带宽限制参数，范围 `[len/2, len*3]`

这里有几个关键点：

1. 每个字段分解都是共享一个IMF参数！实际上仔细想想这就是既不合理又合理。

    - 不合理在：这没有考虑字段之间的不同性，比如有的字段天生就混杂更多的因素，天生需要划分成更多条IMF曲线
    - 合理在：①如果需要为不同字段都单独弄一个指定参数的VMD分解函数，参数量骤增。②只有为每个字段施加了相同的IMF，才能对齐频率、在同一尺度下分析字段间的关系。数据表中计算DTF距离，Pearson和Spearman相关系数才有意义；

2. 正因IMF参数共享，所以IMF参数是全局最优而非某个字段最优。假设某一个字段原始数据是$S$，分解出3个IMF曲线，分别记为$IMF_1、IMF_2$和$IMF_3$，残差记为$\epsilon$，那么：
    $$
    S = IMF_1 + IMF_2 + IMF_3 + \epsilon
    $$
    如果是局部最优，就是直接让字段S的$\mathrm{Spearman}(S, \epsilon)$尽可能小；但这里是全局最优，它的目标是让所有字段算出来的$\mathrm{Spearman}(S, \epsilon)$的平均值最小。

    >此处发现了代码的一点问题，代码中是计算所有字段算出来的$\mathrm{Spearman}(S, \epsilon)$求和后除以总天数，而不是字段数量。虽然不影响实际最终结果，但逻辑上是混乱不正确的。

# 3. 关系图构建

## 3.1 DTW、P和SP系数计算

图是指有实体和边的一种数据结构，在机器学习中常用邻接矩阵来表示。实体就是邻接矩阵各个列的字段，边就是邻接矩阵中的具体数值。论文中明确指出，它并不是简单地仅使用DTW为每个IMF计算一张图，而是同时生成**DTW、Pearson和Spearman**系数的三张图，在模型训练中再对三张图进行融合。

特别的，论文指出DTW本身并不是相似系数，量纲和Pearson和Spearman系数不同，是无法直接作为后续输入的。因此，论文中对DTW计算出的距离使用了如下公式进行归一化：
$$
\text{similarity} = 1 - \frac{\text{DTW距离}}{10 \times \text{序列长度}}
$$
最后，一个IMF文件输出DTW、Pearson和Spearman三张图，一共有足足$3k$张图。形状是`(K, 3, N, N)`，其中的$K$就是VMD中分解的模态数，$N$就是字段数。

## 3.2 稀疏化

稀疏化是指将阈值小于`threshold `的值通通置为0，以及将对角线元素（自指边）置为0。这么做相当于只保留了强相关关系的边，不仅极大简化了图结构，更重要的是能大大降低图卷积的计算量，同时避免了哪些弱关系的边稀释了真正有用的强关系。在代码中，`threshold`取了$0.9$。



## 3.3 Laplacian

Laplacian包含self-loop和对称归一化两个步骤。先说必要性：

- 没加 self-loop（`+I`）：图卷积聚合的是邻居特征，**自己的特征反而没有参与**，每个节点在更新时会忽略自己的信息
- 没有归一化：假设 close 节点和 5 个价格字段都有高相关（度=5），但 adj_close 只和 3 个相关（度=3）。close 的聚合值天然就比 adj_close 大，不是因为它的信息更重要，只是因为它的邻居更多

通过self-loop和归一化，每个节点都获得了和自己相连的权重（self-loop 的 1 被保留），同时边的数值被压缩到相近的量级，避免了度数高的节点主导梯度。

>关于为什么3.2稀疏化时刚把对角线置0，这里在self-loop就又加回来了，乍一看真是扯淡。事实上，如果代码中完全不作对角线置0和self-loop，其实效果和两个都加是一样的。做这一点可能有其他逻辑上的考量？



# 4. 模型构建

## 4.1 简要说明

我们要清楚我们一共需要训练多少个模型。以DJIA为例，一共28支股票，我们需要训练$28k$个模型。也就是说，每一支股票都要单独训练$K$个模型。因为我们将原始信号分解成了$K$个信号 + 噪声，所以我们训练的每一个子模型本质上都是在单独预测某一个分解信号的结果。只有将这$K$个模型的预测结果加起来，才能得到最终的预测结果。这就是论文标题中**"Ensemble"（集成）** 的含义——不是模型层面的 stacking 或 voting，而是对 VMD 分解出的各频率分量分别建模预测，最后求和还原。

另外，GCN、GRU本身都是神经网络架构。我原本误以为这是需要单独训练两个模型进行接力，但实际上这是将两个神经网络架构中的核心设计层给取了出来，共同搭建成一个模型。整个模型在前向传播的过程中要干的事主要有三个：

1. 三图融合，也就是丢进来的一个`(3, N, N)`的图，我们要给他变成`(N, N)`的图，这是第一层干的事；
2. GCN图卷积：把所有字段按照它们在图上的关系相互加权融合，然后仅取**收盘价**这个节点的聚合数据，再投影到隐藏空间，喂给 GRU；
3. GRU门控：把这个时间步的特征和上一个时间步的记忆 `h` 结合，决定"记住什么、忘记什么、输出什么"



## 4.2 三图融合

```python
# wa:   (3, n, n)  ← 图类型级别的可训练权重
# adj:  (3, n, n)  ← 经过 Laplacian 的 P/S/D 图

integrated_inputs = wa * adj     # (3, n, n)  逐元素乘
adj_max = sum(integrated_inputs, axis=0)  # (n, n)  三张图求和
```

这里只需要注意一个点：`wa`权重矩阵的维度是`(3, n, n)`的，这相当于对字段 i 和字段 j 之间的边（比如 open→close），三张图各自贡献一个值，各自乘以一个可训练权重，求和得到最终进入 GCN 的边权。



## 4.3 GCN图卷积

**① 消息传递：** `(batch, n) × (n, n) → (batch, n)`

每一列的字段值 = 所有字段值的加权求和，权重由邻接矩阵决定：

```
gc_output[open]   = adj[open,open]*open + adj[open,close]*close + ... + adj[open,vol]*vol
gc_output[close]  = adj[close,open]*open + adj[open,close]*close + ... + adj[open,vol]*vol
...
```

这相当于每个字段都"看到"了其他字段的信息，相互加权融合。**这就是 GCN 中最核心的"图卷积"操作。**

**② 收盘价特征提取：** `(batch, n) → (batch,)`

我们只关心收盘价的预测，所以把其他字段的聚合结果扔掉，只取第 `s_index=3` 列。这一步没有可训练参数，是硬编码的。

**③ 维度扩展：** `(batch,) → (batch, 1)`

给 GRU 做准备，加一个特征维度。

**④ 投影：** `(batch, 1) × (1, units) → (batch, units)`

把 1 维的收盘价聚合特征映射到 GRU 的隐藏空间（`n_neurons`），这样 GRU 门控才能和隐状态做矩阵乘法。



## 4.4 GRU门控

```python
# GRU
z = K.dot(x, self.wz) + K.dot(x, self.uz) + self.bz    # 更新门
z = sigmoid(z)
r = K.dot(x, self.wr) + K.dot(x, self.ur) + self.br    # 重置门
r = sigmoid(r)
h = K.dot(x, self.wh) + K.dot((r * state), self.uh) + self.bh  # 候选隐状态
h = tanh(h)

output = z * state + (1 - z) * h    # 最终隐状态
```

**重置门 `r`**：决定"要不要忘掉过去"。值接近 0 → 几乎完全忽略历史隐状态，只依赖当前输入；值接近 1 → 完整保留历史。`r * state` 就是按比例擦除记忆。

**更新门 `z`**：决定"新信息替换旧信息的比例"。值接近 1 → 几乎完全保留旧状态；值接近 0 → 几乎完全用新状态替换。

**候选隐状态 `ĥ`**：用重置门处理过的历史信息，结合当前输入，生成一个"候选记忆"。

**最终隐状态 `h_t`**：在旧状态和候选状态之间做插值：`z * old + (1-z) * candidate`。



## 4.5 核心损失函数

论文区别于基线模型的关键创新：在 MSE 基础上引入**趋势准确率正则化**，平衡数值精度与涨跌方向预测。纯 MSE 只关心"数值离真实值有多近"，不关心"涨跌方向对不对"。股票预测中方向比数值更关键——预测值差 0.01 但方向对了能赚钱，方向错了会亏钱。纯 MSE 模型倾向于输出"平滑但滞后"的预测曲线，拐点全错。

以下是损失函数的严谨定义：
$$
\mathcal{L}_{\text{total}} = \underbrace{\frac{1}{n}\sum(y-\hat{y})^2}_{\text{MSE 数值精度}} \;+\; \lambda_1\underbrace{\|w\|_1}_{\text{L1 稀疏}} \;+\; \lambda_2\underbrace{\frac{1}{n}\sum\mathbb{1}[\text{sign}(\Delta y)\neq\text{sign}(\Delta \hat{y})]}_{\text{方向错了就罚}}
$$

- $\Delta y = y_t - y_{t-1}$：日间涨跌
- $\mathbb{1}[\cdot]$：方向预测错误时为 1
- $\lambda_1 = 0.01$（L1 系数），$\lambda_2 = 0.1$（趋势系数）—— 论文实验确定

论文 Table 6 Method 10（去掉趋势项、回退到纯 MSE）显示：ACC 从 0.6940 降至 0.6795、RMSE 从 0.1500 升至 0.1595，验证了趋势正则化的有效性。

> 当前代码（REGCN.py）中**未实现上述自定义损失函数**，仅用了默认 MSE，论文宣称的趋势准确率优势无法复现。
