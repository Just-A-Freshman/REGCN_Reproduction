# Mistakes Log

## 2026-06-19 — DJIA 滑动窗口消融

1. **花 2 小时做 SFTP 上传，但数据早已在远端** — 用户已同步好项目，我只管执行代码就行。浪费大量时间在传输上。

2. **Shell heredoc 变量展开搞砸** — 用 `<<EOF` 没加引号导致 `${sl}` 被 bash 展开为空。应该用 `<<"EOF"` 或 `<<'EOF'`。

3. **Python `\\n` 当换行用** — 在 Python 三引号字串里写 `\\n` 结果写在文件里是字面 `\n` 而不是真正换行。配置文件内容全挤在一行。应该直接用真实换行，或逐行 write。

4. **嵌套 f-string Python 3.10 不支持** — 写了 `f"xxx {len(open(f'path/{x}.csv').readlines())}"` 这种嵌套 f-string，Python 3.10 报语法错误。改用旧式 `%` 格式化。

5. **贪心优化并行度，不先验证基础可用** — 先试 n_jobs=28 卡住，又试 n_jobs=14 还是慢，最后 n_jobs=1 反而稳定。应该先确保能跑通再优化。

6. **脚本越套越复杂** — base64 → heredoc → Python subprocess → nohup 多层嵌套，出问题根本找不到根因。应该用最简单的单层方式。

7. **不先检查远端现状就直接操作** — 没确认 config.ini 当前状态就运行 sed，导致 seq_len 被改空值。应该先 cat 确认再动手。

## 规则

- **远端数据只读，不改。** 需要运行代码就直接执行，不要重新上传数据文件。
- **先跑通，再优化。** 先 n_jobs=1 验证基础流程，再考虑并行加速。
- **远端执行用最简单的方式。** 能用 Python 解决的不用 shell 脚本，避免 shell 转义问题。
- **每次改动前先检查远端状态。** `cat` 一下当前文件内容再动手。
