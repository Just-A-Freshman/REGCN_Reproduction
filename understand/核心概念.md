# 我刚弄懂的

## 简单概念

**均方根误差（RMSE）** ，核心特点，对异常值惩罚更重（**开方操作只是改变了最终误差的尺度（从平方尺度拉回线性尺度），但平方并平均后，各个误差对最终结果的相对贡献比例已经被永久改变了。**），形式上很像标准差，只是用来计算拟合效果了：
$$
\sqrt{\frac{1}{n}\sum_{i=1}^n (y_i - \hat{y_i})^2}
$$

**交易数据**：股票市场每天产生的基础数据，核心指6项数据：

- 开盘价格、收盘价格
- 最高价、最低价
- 成交量、成交额



**多变量时间序列**：不是只看一个数字，而是同时看好几个。比如同时看开盘价、收盘价、最高价、最低价、成交量，这些一起组成了"多变量"。



## 变分模态分解（VMD）

真实的股票数据就是一个波形极为复杂的曲线。这一条线里其实是**多种不同节奏的波叠在一起**，如果直接让机器学习，由于里面混杂了太多要素，机器很难真正学会。VMD 就是把这混合的一条线拆成几条**干净的子波**（模态/本征模态函数（IMF） IMF），每条只包含某一个节奏。例如：

```
原始股价 → [ VMD 算法 ] → IMF₁ (慢波、大趋势)
                       → IMF₂ (中速波、行业周期)
                       → IMF₃ (快波、短期波动)
                       → 残差 (剩余噪声)
```

VMD 把问题当作一个**约束优化**来处理。核心想法：在所有可能的拆分方案中，找到一个让**每种子波都"最窄"**（集中在某个频率附近，即频率稳定）的方案，同时保证合起来能**还原原信号**。具体算法不作深入了解。



## 图、节点、边
以朋友圈为例子：
- **节点（Node / Vertex）**：就是**每个人**。比如你、你的好朋友小明、你的同学小红，每个独立的人就是一个节点。
- **边（Edge）**：就是**人与人之间的关系**。比如“你和小明是好友”，这就是一条连接你们两个节点的边。
- **图（Graph）**：就是**由所有这些人和他们之间关系组成的整个网络**。

由边的类型可以定义不同的图：

- 如果“好友关系”是双向的（你加我，我也加你），这叫**无向图**。
- 如果是在微博上，“你关注了小红”，但小红没有回关你，这种单向关系就叫**有向图**，边会带上箭头。
- 如果给关系加上强弱，比如“你和小明每周通话3次”，这个“3”就是边的**权重**，这样的图叫**加权图**，可以用来计算谁是你最亲密的朋友。

切记，节点只是一个抽象的实体标识，它可以是任意类型。由节点是否是同一类型也可以定义不同的图：

- 如果节点都是同一类型，比如同时人，都是城市，就叫它“**同构**信息网络”；
- 如果节点存在不同类型，这样的图被称为**异构**信息网络；



## 节点嵌入

即将图结构（节点 + 边）中的节点转化为向量。在嵌入计算的过程中，使用边的信息迭代节点的向量表示。一般来说：

- 边的信息直接蕴含在向量与向量之间的空间信息中。
- 但同时，在一些更复杂的模型中（如知识图谱的TransE、RotatE），关系本身也被直接表示为一个向量。他们会训练这个向量。

需要知道，原始的图结构有着清晰的语义，比如A是B的同学，但转为向量后，我们只能看到A和B在空间中关系很近。这种转化是有损的，将清晰的语义性质转化成了定量的数字描述。



## 图卷积神经网络（GCN）

- 是什么：一种从图结构中学习的神经网络。输入是包含节点和边信息的图，输出是**节点嵌入向量**。
- 为什么：将图结构转为节点嵌入向量有三大应用目的。
    - 节点分类：对新输入的节点划定标签。具体而言，我们用训练好的节点嵌入向量当作**特征向量 X**，把已知的标签当作**目标 y**，训练一个分类器；
    - 链接预测：预测两个新输入的节点之间是否存在（或即将出现）一条边。具体而言：
        - 我们会定义一个**评分函数**，计算两个节点向量之间“存在边”的可能性。常见的有：
            - **点积**：`score(u, v) = embedding(u) · embedding(v)`
            - **余弦相似度**：就是向量的夹角余弦。
            - **更复杂的边预测网络**：把两个向量拼起来，送入一个小型神经网络，输出一个分数。
        - 训练时，我们拿图中**已经存在的边**作为正样本（标签=1），再随机采样一些**没有边的节点对**作为负样本（标签=0），让模型学习什么样的向量关系对应着有边。
        - 预测时，对于所有还没有边的节点对，计算分数，分数最高的几个，就是最可能产生新链接的候选。
    - 图分类：本质是节点分类的变体。它的核心动作是将图也转化成向量。具体来说：第一步要在一个图内部训练向量表示（**消息传递**）；第二步把整个包含多个向量的图再转化成向量（**图读出**）。想象一个班级（图）要竞选“最佳班级”（图分类），第一步就是根据一个班级中同学与同学之间的关系，构建出同学向量。向量的个数取决于这个班级的同学个数。第二步就是将这所有同学向量汇总成一个图向量。常见的汇总方法包括：”平均法“、”求和法“和”注意力法“（将代表性学生向量加权）。
- 怎么做：**利用边的信息**来学习节点的嵌入向量。



## GRU神经网络

循环神经网络的升级版。特别擅长处理时间序列数据。

|          | RNN                    | GRU                            |
| -------- | ---------------------- | ------------------------------ |
| 记忆机制 | 每次都无脑混合新旧数据 | 有专门的闸门控制该记该忘       |
| 长程记忆 | 差，传几轮就没了       | 好，更新门可以长期保持某个记忆 |
| 参数数量 | 少                     | 多一点（多两组权重）           |
| 效果     | 短序列还行，长序列不行 | 长短序列都好                   |

GRU的核心设计：

- 重置门（Sigmoid）：决定丢掉哪些旧记忆
- 候选隐藏状态：用筛选过的旧记忆和新输入，生成“新点子”
- 更新门（Sigmoid）：决定筛选过的旧记忆 和 新点子 的混合比例：` z × 旧记忆 + (1 − z) × 新点子`



## 时空关系

时空关系是时间关系和空间关系的组合。

先说空间关系。它是指同一时刻，不同数据项之间的关系。比如同一天中：

- 如果今天**开盘价**就很高，**收盘价**通常也不会太低；
- 如果**成交量**突然放大，价格通常会有大波动；

再说时间关系。它是指同一个数据项，在不同时刻中的关系。比如对于股价这个数据项：

- 昨天跌了 5%，今天继续跌的概率不小（趋势延续）
- 连续涨了 5 天，可能明天要回调（趋势反转）



## 遗传算法（GA）

- 是什么：一种模仿"物竞天择"的优化方法。把一堆可能的参数丢进去，让它们像生物进化一样交配、变异，经过多代筛选，留下表现最好的那个**参数组合**。
- 怎么做：迭代过程？
    - 第一步，随机生成候选参数并尝试；
    - 第二步，评估/计算适应度（打分），选出好的参数；
    - 第三步，变异或交叉得到后代，并回到第二步。变异就是在好参数附近微调探索，交叉就是将几个好参数的局部组合起来试一下；
- 为什么：
    - 凭什么交叉会变得更好：实际上交叉不一定会更好，只是通常而言，我们用好参数的局部拼接在一起，变好的概率更大。
    - 后代变差了怎么办：首先建立一个观念，后代变差不一定是坏事，它可能是在跳出局部最优解后短暂的变差；其次，遗产算法一般采用精英保留策略，即始终让部分非常优秀的参数不经过变异或交叉就传递给下一代，保证好的种子能延续；



## DTW和FastDTW

先说DTW：

- 是什么：DTW（Dynamic Time Warping）是用来计算两组时间序列数据相似性的方法。注意这个相似性是一种**类距离**，而不是处于0-1之间的值。所谓类距离，是因为它不满足距离的对称性，也就是A到B的DTW相似度不等于B到A的DTW相似度。
    - 传统的算法需要时间长度的严格对齐，并且是同一时刻一一对应的计算距离。这不仅导致了很多时间序列数据无法计算相似度，而且会让两个原本趋势相近的时间序列，可能因为位置偏差导致计算出过大的差距。
    - DTW允许"一对多"和"多对一"的对齐，不仅能不同长的时间序列也可以计算相似度，更为关键的是，它的对齐算法更能捕获两个时间序列的一致性趋势的一致性。
- 为什么：DTW的核心产出就是这个距离，它的用途很广泛。在股票中的应用例子：
    - **构建股票图**：用 DTW 算两只股票的走势相似度，如果走势高度同步（涨一起涨、跌一起跌），说明这两支股票大概率存在着某种内在联系，于是我们可以在两支股票之间构建一条权重边。这样一来，预测股票涨跌时，利用这张股票图谱，我们可以充分利用其他股票的信息。
    - **模式匹配预测**：把今天的走势模式和历史上的某段模式做 DTW 匹配。如果过去出现类似走势后股价涨了，那今天也可能涨。这就是所谓的"模式匹配预测"；
    - **股票聚类**：把所有股票的走势两两算 DTW 距离，然后聚类
- 怎么做：详情参考[理解DTW](https://zhuanlan.zhihu.com/p/117634492)



再说FastDTW：

显然，它就是DTW的改进算法。假设两个时间序列的长度分别为$m$和$n$。那么，传统的DTW算法时间复杂度$O(m \cdot n)$，而FastDTW在几乎不影响其最终结果的情况下将时间复杂度降到了线性级别。



其他问题（关键）：

1. DTW算出来的只是距离，这个数字和序列总长度是相关的。序列越长，DTW越大。是不是应该有某种归一化的方法呢？ 
    回答：的确是要归一化的，有两种常见的归一化方法：
    - **除以路径长度** — 最简单。DTW 最后找到的最优对齐路径长度（即走了多少步），用它来除总距离，得到"平均每步的扭曲代价"。不管序列多长，这个值在同一个尺度上。
    - **最大归一化** — 先算一个理论最大可能 DTW 距离（全序列最极端的错位情况），然后用实际 DTW ÷ 最大 DTW，得到 0-1 之间的相似度。
2. 对于两种模式恰好相反的情况，DTW好像完全没办法捕捉这种相反的相似性啊。
    回答：这的确是DTW固有的不足。因此我们**不只用 DTW，而是三个指标一起用：**DTW归一化后的相关系数 + Pearson相关系数 + Spearman相关系数。需要注意，因为Pearson系数和Spearman相关系数的计算方式，天然就要求了两个时间序列必须是等长的，不能错位。如何融合三个指标？本篇论文的思路是给三个指标一个可学习的权重，然后用深度学习算法学习并固化这个权重。





# 我本身就懂的

**股票价格预测** — 就是猜明天股票是涨还是跌（方向）、会到多少钱（具体数值）。

**非线性** — 简单说就是"不是一条直线"。股票走势不是直上直下的，中间有很多弯弯绕绕，很难用一个简单公式来描述。

**非平稳性** — 数据的规律会变。比如这周的波动规律到了下周就不管用了，均值、方差都在随时间漂移，不稳定。

**时间序列** — 一串按时间顺序排列的数据点，比如 1 月 1 日的股价、1 月 2 日的股价、1 月 3 日的股价……一串下来就是一条时间序列。

**噪声** — 数据里的"杂音"，没有意义的随机波动。就像收音机里的沙沙声，干扰你听清楚内容。股价数据里噪声特别多。

**正则化** — 在模型训练时加一点"惩罚"，防止模型学得太死板（过拟合）。就像考试时要求解题不仅答案对还要步骤简洁，防止学生死记硬背。

**L1 正则化** — 一种"惩罚"方式，它会让模型里不重要的参数直接变成 0，起到简化模型的作用。就像清理衣柜——不穿的衣服直接扔掉而不是叠起来放好。

**皮尔逊相关系数** — 衡量两个东西之间"线性关系"的强弱，范围在 -1 到 1 之间。1 表示一个涨另一个也同步涨，-1 表示一个涨另一个就跌，0 表示没啥关系。比如"开盘价"和"收盘价"的皮尔逊系数通常接近 1。

**斯皮尔曼等级相关系数** — 和皮尔逊类似，但它衡量的是"单调关系"（不要求是直线），而且用排名代替原始数值。更耐用一些，不容易被极端值带偏。

**集成学习** — 把多个模型的结果合并起来做最终决策。比如问 10 个专家分别做预测，最后把他们的意见汇总，通常比单独一个专家更靠谱。

**准确率（ACC）** — 预测正确的比例。比如预测了 100 天走势，70 天方向猜对了，ACC = 70%。

**平均绝对误差（MAE）** — 也是衡量预测值和真实值的差距，但它直接取绝对值平均，没有平方那一步。比 RMSE 更直观，不会因为一个特别大的错误就被拉到很高。
