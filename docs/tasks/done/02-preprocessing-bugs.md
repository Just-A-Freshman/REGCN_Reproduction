# 预处理 Bug：GA 适应度计算错误 + 数据泄露

## B3：GA 适应度除 BUG

**文件：** `dataprecossing/GA_VMD.py` `Fun()`

**问题：** 最后一行 `s /= data.shape[1]` 除以的是**序列长度**（约 601），应该除以**字段数**（SSE=9, DJIA=6）。

**修复：** 将 `s /= data.shape[1]` 改为 `s /= data.shape[2]`。注意 data 的形状为 `(batch, timesteps, features)`，`shape[2]` 才是字段数。

**验证：** 打印修复前后的适应度值，数值应不同但 GA 选出的最优参数一致（因为 selection 只比较相对大小）。

## B4：图构建数据泄露

**文件：** `dataprecossing/adjprocessing.py`

**问题：** Pearson / Spearman / DTW 相关性计算使用了全量数据（训练+测试），测试集信息提前暴露给图结构。这违反了时间序列预测的基本约束。

**影响：** 模型评估结果可能偏高，不能反映真实泛化能力。

**修复思路：**
1. 相关性计算（Pearson / Spearman / DTW）改为仅使用训练子集
2. 图结构（邻接矩阵）的维度仍是 N×N（字段数），只影响边权的具体数值
3. 测试集和验证集使用相同的图结构，但图是从训练数据计算得出的

**验证：** 对比修复前后的邻接矩阵数值差异。图结构（拓扑）应基本一致，但边权数值应不同。
